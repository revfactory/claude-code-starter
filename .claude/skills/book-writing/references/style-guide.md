# 도서 스타일 가이드

## 문체
- **경어체**: "~합니다", "~됩니다"
- **호칭**: 독자를 "여러분"으로 호칭
- **톤**: 친근한 멘토가 옆에서 설명하는 느낌
- **길이**: 한 문장은 40자 이내 권장, 최대 60자

## 용어 통일표

| 통일 용어 | 사용 금지 | 비고 |
|----------|----------|------|
| 프롬프트 | 명령어, 질문, 입력 | Claude Code에 보내는 텍스트 |
| 도구(Tool) | 기능, 함수 | Claude Code의 내장 도구 |
| 워크스페이스 | 작업 공간, 작업 디렉토리 | 프로젝트 루트 |
| 컨텍스트 | 맥락, 문맥 | AI가 참조하는 정보 |
| 슬래시 커맨드 | 슬래시 명령, 명령어 | /로 시작하는 명령 |
| 훅(Hook) | 후크 | 이벤트 기반 자동 실행 |
| MCP 서버 | MCP | 항상 "MCP 서버"로 풀어쓰기 |

## 포맷 규칙

### 코드블록
```
코드블록에는 반드시 언어 태그를 붙인다
예: ```bash, ```python, ```json
```

### 팁/주의 박스
```markdown
> **팁**: 유용한 추가 정보
> **주의**: 실수하기 쉬운 부분
> **참고**: 배경 지식이나 심화 내용
```

### 챕터 구조
1. 챕터 제목
2. 학습 목표 (이 챕터에서 배울 것)
3. 본문 (개념 설명 + 실습)
4. 정리 (핵심 요약)
5. 연습 문제 (선택)

### CLI 대화 예시 포맷
```
$ claude "프롬프트 내용"

[Claude의 응답 요약 또는 전문]
```

## 분량 가이드
- 챕터당 3,000~5,000자 (A4 5~8페이지)
- 전체 도서 15~20챕터 목표
- 코드 예제 포함 시 코드는 분량에서 제외
