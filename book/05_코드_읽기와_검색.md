# 챕터 5: 코드 읽기와 검색

> **학습 목표**
> - Claude Code에게 코드를 설명받을 수 있다
> - 파일 검색(Glob)과 코드 검색(Grep)의 차이를 이해한다
> - 대규모 코드베이스에서 효율적으로 탐색할 수 있다

---

새로운 프로젝트에 투입되었다고 상상해보세요.
수백 개의 파일, 낯선 구조, 긴 함수들.
어디서부터 읽어야 할까요?

Claude Code는 여러분의 코드 내비게이터입니다.
"이게 뭐야?"라고 물으면 됩니다.
이번 챕터에서는 코드를 읽고 검색하는
세 가지 핵심 방법을 배워보겠습니다.

---

## 5.1 코드 설명 요청하기

가장 직관적인 방법부터 시작합니다.
그냥 자연어로 물어보세요.

### 함수 단위 질문

```bash
$ claude "src/utils/validate.ts의 validateEmail 함수가 무엇을 하나요?"
```

```
[응답]
validateEmail 함수는 이메일 주소의 유효성을 검사합니다.

1. 정규식으로 기본 형식을 확인합니다
2. 도메인 부분에 점(.)이 포함되어 있는지 검사합니다
3. 유효하면 true, 아니면 false를 반환합니다
```

Claude Code는 해당 파일을 직접 읽고 분석합니다.
코드를 복사해서 붙여넣을 필요가 없습니다.

### 전체 아키텍처 파악

프로젝트 전체를 파악하고 싶을 때도 있습니다.

```bash
$ claude "이 프로젝트의 전체 구조를 설명해줘"
```

Claude Code는 디렉토리를 탐색하고,
주요 파일들을 읽은 뒤 구조를 정리해줍니다.
`package.json`, `tsconfig.json` 같은
설정 파일도 함께 분석합니다.

더 구체적으로 질문할수록 좋습니다.

```bash
$ claude "src/api/ 디렉토리의 라우팅 구조를 설명해줘"
```

> **팁**: `@파일경로`를 사용하면 특정 파일을 직접 컨텍스트에 포함할 수 있습니다. 예를 들어 `@src/index.ts 이 진입점에서 시작해서 앱 구조를 설명해줘`처럼 쓸 수 있습니다.

## 5.2 파일 찾기 — Glob 도구

"어딘가에 설정 파일이 있는데..."
파일 이름이나 확장자는 알지만,
정확한 위치를 모를 때 Glob이 빛납니다.

### Glob이란?

Glob은 **파일 이름 패턴**으로 파일을 찾는 도구입니다.
ripgrep이 아닌 파일 시스템 패턴 매칭을 사용합니다.
결과는 수정 시간순으로 정렬됩니다.

### 자주 쓰는 Glob 패턴

| 패턴 | 의미 |
|------|------|
| `**/*.ts` | 모든 하위 폴더의 TypeScript 파일 |
| `src/**/*.tsx` | src 아래 모든 React 컴포넌트 |
| `**/test/**/*.spec.js` | 테스트 폴더의 스펙 파일 |
| `**/*.config.*` | 모든 설정 파일 |
| `src/**/index.ts` | src 아래 모든 index 파일 |

### 실전 예제

```bash
$ claude "이 프로젝트에서 모든 설정 파일을 찾아줘"
```

```
[응답]
설정 파일을 검색했습니다.

Glob 패턴 **/*.config.* 으로 검색한 결과:
- tailwind.config.js
- jest.config.ts
- vite.config.ts
- eslint.config.mjs

추가로 발견한 설정 파일:
- tsconfig.json
- package.json
- .env.example
```

Claude Code는 상황에 맞는 패턴을 스스로 선택합니다.
여러분은 "무엇을 찾고 싶은지"만 말하면 됩니다.

> **팁**: Glob은 파일 **이름과 경로**로 검색합니다. 파일 **내용**을 검색하려면 다음 섹션의 Grep을 사용하세요.

## 5.3 코드 검색 — Grep 도구

"이 함수를 어디서 호출하지?"
"TODO 주석이 몇 개나 남아 있지?"

파일 내용 속 특정 텍스트를 찾을 때는
Grep 도구를 사용합니다.

### Grep이란?

Grep은 **파일 내용**에서 패턴을 검색하는 도구입니다.
내부적으로 ripgrep을 사용하여 매우 빠릅니다.
정규식도 지원합니다.

### Glob vs Grep 비교

| 구분 | Glob | Grep |
|------|------|------|
| 검색 대상 | 파일 **이름/경로** | 파일 **내용** |
| 사용 시점 | "파일이 어디 있지?" | "이 코드가 어디 있지?" |
| 패턴 종류 | 와일드카드 (`*`, `**`) | 정규식 (`.*`, `\w+`) |
| 예시 | `**/*.ts` | `function validate` |

### 실전 예제

```bash
$ claude "프로젝트에서 TODO 주석을 모두 찾아줘"
```

```
[응답]
TODO 주석을 검색했습니다. 총 4개 발견:

src/api/auth.ts:42
  // TODO: refresh token 로직 추가

src/utils/date.ts:15
  // TODO: timezone 처리 개선

src/components/Header.tsx:28
  // TODO: 반응형 메뉴 구현

src/db/migration.ts:7
  // TODO: 롤백 로직 구현
```

정규식을 활용한 고급 검색도 가능합니다.

```bash
$ claude "console.log가 남아있는 곳을 모두 찾아줘. 테스트 파일은 제외해줘"
```

Claude Code는 Grep 도구에 적절한 필터를 적용하여
테스트 파일을 제외한 결과만 보여줍니다.

> **주의**: 대규모 프로젝트에서 너무 일반적인 패턴(예: `the`, `import`)을 검색하면 결과가 매우 많을 수 있습니다. 가능한 구체적인 패턴을 사용하세요.

## 5.4 Plan 모드 활용하기

코드를 탐색할 때 실수로 파일을 수정하면 곤란합니다.
Plan 모드는 이 걱정을 없애줍니다.

### Plan 모드란?

Plan 모드는 **읽기 전용** 모드입니다.
파일을 읽고 검색할 수 있지만,
수정이나 명령 실행은 할 수 없습니다.

### Plan 모드 진입 방법

프롬프트 입력 전에 `Shift+Tab`을 두 번 누르세요.
프롬프트 앞에 `(plan)` 표시가 나타납니다.

```
(plan) > 이 프로젝트의 인증 흐름을 분석해줘
```

```
[응답]
인증 흐름을 분석했습니다.

1. 사용자가 /api/auth/login에 POST 요청
2. auth.controller.ts에서 요청을 받음
3. auth.service.ts에서 비밀번호 검증
4. JWT 토큰 생성 후 반환
5. 미들웨어(auth.guard.ts)에서 토큰 검증

관련 파일:
- src/api/auth/auth.controller.ts
- src/api/auth/auth.service.ts
- src/middleware/auth.guard.ts
- src/utils/jwt.ts
```

Plan 모드에서는 Claude Code가 자유롭게 탐색하되,
어떤 파일도 건드리지 않습니다.
낯선 코드베이스를 처음 파악할 때 안성맞춤입니다.

> **팁**: Plan 모드에서 탐색한 뒤, 수정이 필요하면 일반 모드로 돌아가서 "아까 분석한 인증 흐름에서 토큰 갱신 로직을 추가해줘"처럼 이어서 요청할 수 있습니다.

## 5.5 실습: 낯선 프로젝트 파악하기

지금까지 배운 기술을 종합해봅시다.
오픈소스 프로젝트를 클론해서
5분 안에 구조를 파악하는 실습입니다.

### 단계별 실습

**1단계: 프로젝트 시작**

```bash
$ git clone https://github.com/expressjs/express.git
$ cd express
$ claude
```

**2단계: Plan 모드로 전체 구조 파악**

```
(plan) > 이 프로젝트의 전체 구조와 핵심 모듈을 설명해줘
```

Claude Code가 디렉토리를 탐색하고
주요 파일을 읽어 구조를 설명해줍니다.

**3단계: 핵심 파일 찾기**

```
(plan) > 라우팅을 처리하는 핵심 파일들을 찾아줘
```

Glob 도구로 파일 위치를 찾고,
Grep 도구로 라우팅 관련 코드를 검색합니다.

**4단계: 특정 기능 추적**

```
(plan) > app.get()을 호출하면 내부에서 어떤 일이 일어나는지 추적해줘
```

**5단계: 의존성 파악**

```
(plan) > 외부 의존성 목록과 각각의 역할을 알려줘
```

### 실습 요약 체크리스트

- [ ] 전체 디렉토리 구조를 파악했다
- [ ] 핵심 모듈과 진입점을 찾았다
- [ ] 특정 기능의 코드 흐름을 추적했다
- [ ] 외부 의존성을 확인했다

이 네 단계를 거치면 어떤 프로젝트든
빠르게 전체 그림을 그릴 수 있습니다.

## 정리

이번 챕터에서 배운 내용을 정리합니다.

| 방법 | 도구 | 사용 시점 |
|------|------|----------|
| 코드 설명 요청 | Read | "이 코드가 뭘 하는 거야?" |
| 파일 찾기 | Glob | "이 파일이 어디 있지?" |
| 코드 검색 | Grep | "이 패턴이 어디에 쓰이지?" |
| 안전한 탐색 | Plan 모드 | "수정 없이 분석만 하고 싶어" |

핵심은 간단합니다.
**자연어로 물어보세요.**
Claude Code가 적절한 도구를 알아서 선택합니다.

다음 챕터에서는 코드를 읽는 것에서 한 발 나아가,
**코드를 수정하고 생성하는 방법**을 배워보겠습니다.
