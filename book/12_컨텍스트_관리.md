# 챕터 12: 컨텍스트 관리의 기술

> **학습 목표**
> - 컨텍스트 윈도우의 5개 영역을 이해한다
> - /context로 토큰 분포를 진단할 수 있다
> - /compact와 /clear를 전략적으로 사용한다
> - 긴 세션에서 컨텍스트를 효율적으로 운용한다

---

## 12.1 컨텍스트 윈도우 해부

Claude Code의 컨텍스트 윈도우는 유한합니다. 여러분이 보내는 모든 메시지, 도구 출력, 시스템 지시가 이 공간을 차지합니다. 구조를 이해하면 효율이 달라집니다.

### 5개 영역

컨텍스트 윈도우는 다음 5개 영역으로 구성됩니다.

![컨텍스트 윈도우 구조](images/ch12_context_window.png)

**영역별 특성을 정리합니다.**

- **① 시스템 프롬프트**: Claude의 기본 행동을 정의합니다. 여러분이 수정할 수 없습니다.
- **② CLAUDE.md**: 프로젝트 규칙과 자동 메모리입니다. 매 요청마다 전송됩니다.
- **③ 스킬 설명**: 등록된 스킬의 이름과 설명입니다. 스킬이 많을수록 공간을 차지합니다.
- **④ MCP 도구 스키마**: 연결된 MCP 서버의 도구 정의입니다. 서버가 많으면 부담이 커집니다.
- **⑤ 대화 이력**: 여러분의 메시지와 응답입니다. 가장 많은 공간을 차지합니다.

> **주의**: ①~④는 매 요청마다 반복 전송됩니다. CLAUDE.md가 500줄이면 매번 그만큼 토큰을 소모합니다. 200줄 이하를 유지하세요.

## 12.2 /context 진단: 토큰 분포 확인

현재 컨텍스트 상태가 궁금하다면 `/context`를 실행하세요.

```bash
> /context
```

실행하면 다음과 같은 분포를 확인할 수 있습니다.

```
Context window usage: 45,230 / 200,000 tokens (22.6%)

  System prompt:     3,200 tokens  (1.6%)
  CLAUDE.md:         1,800 tokens  (0.9%)
  Skills:              640 tokens  (0.3%)
  MCP tools:         2,100 tokens  (1.1%)
  Conversation:     37,490 tokens  (18.7%)
```

**진단 포인트 3가지를 기억하세요.**

1. **CLAUDE.md 비율**: 5% 이상이면 내용을 줄이세요
2. **MCP 도구 비율**: 3% 이상이면 미사용 서버를 끊으세요
3. **대화 비율**: 80%를 넘으면 곧 자동 압축이 발생합니다

> **팁**: 작업 중간중간 `/context`로 점검하는 습관을 들이세요. 갑작스러운 자동 압축으로 맥락을 잃는 것을 방지합니다.

## 12.3 /compact 전략

컨텍스트가 차오르면 `/compact`로 압축합니다. 두 가지 방식이 있습니다.

### 전체 압축

```bash
> /compact
```

대화 전체를 요약합니다. 범용적이지만 세부 내용이 손실될 수 있습니다.

### 주제 집중 압축

```bash
> /compact focus on API authentication changes
```

특정 주제를 중심으로 압축합니다. 핵심 맥락을 보존하면서 불필요한 부분을 제거합니다.

**어떤 방식을 선택할까요?**

| 상황 | 추천 방식 |
|------|-----------|
| 여러 작업을 넘나든 경우 | 전체 압축 |
| 하나의 기능을 깊이 구현 중 | 주제 집중 압축 |
| 디버깅 중간에 컨텍스트 부족 | 주제 집중 압축 |
| 대용량 파일 출력이 쌓인 경우 | 전체 압축 |

> **팁**: 주제 집중 압축의 지시는 영어가 효과적입니다. 내부적으로 영어 프롬프트로 처리되기 때문입니다.

## 12.4 언제 /clear할까

`/compact`가 "정리"라면 `/clear`는 "초기화"입니다. 대화 이력을 완전히 삭제합니다.

```bash
> /clear
```

**`/clear`가 적합한 순간은 다음과 같습니다.**

- **새로운 작업 시작**: 인증 구현 후 UI 작업으로 전환할 때
- **주제가 완전히 바뀔 때**: 백엔드에서 프론트엔드로 이동
- **컨텍스트 오염**: 잘못된 맥락이 응답에 영향을 줄 때
- **85% 이상 사용**: 압축보다 초기화가 깔끔할 때

**`/compact` vs `/clear` 판단 기준입니다.**

| 기준 | /compact | /clear |
|------|----------|--------|
| 이전 맥락 필요 | O | X |
| 같은 작업 계속 | O | X |
| 완전히 새 작업 | X | O |
| 토큰 80% 초과 | 가능 | 권장 |

> **주의**: `/clear` 후에는 이전 대화를 참조할 수 없습니다. 중요한 결정사항은 `/clear` 전에 CLAUDE.md에 기록하세요.

## 12.5 긴 세션 전략

대규모 리팩토링이나 장시간 작업에서는 컨텍스트 관리가 생존 기술입니다.

### 자동 압축 설정

컨텍스트가 95%에 도달하면 자동 압축이 실행됩니다. 이 임계값을 조정할 수 있습니다.

```bash
# 80%에서 자동 압축 시작
export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=80
```

낮게 설정하면 더 자주 압축됩니다. 맥락 손실이 걱정되면 70~80%를 추천합니다.

### 확장 사고 활용

복잡한 작업에서 사고 토큰을 늘릴 수 있습니다.

```bash
# 사고 토큰 예산 설정
export MAX_THINKING_TOKENS=20000
```

> **주의**: 사고 토큰도 컨텍스트를 소모합니다. 필요한 작업에만 설정하세요.

### 1M 토큰 모드

`opus` 모델은 최대 1M 토큰을 지원합니다. 대규모 코드베이스 분석에 유용합니다.

```bash
# 모델 변경으로 확장 컨텍스트 사용
> /model opus
```

### 긴 세션 체크리스트

장시간 작업 시 다음 루틴을 따르세요.

1. **시작 전**: CLAUDE.md에 작업 목표를 기록합니다
2. **30분마다**: `/context`로 사용량을 확인합니다
3. **50% 도달**: 중간 성과를 CLAUDE.md에 기록합니다
4. **70% 도달**: `/compact`로 주제 집중 압축합니다
5. **작업 전환**: `/clear` 후 새 세션을 시작합니다

> **팁**: 서브에이전트(Agent 도구)를 활용하면 대용량 출력을 격리할 수 있습니다. 메인 컨텍스트에는 결과 요약만 남습니다.

## 정리

| 개념 | 핵심 요약 |
|------|-----------|
| 컨텍스트 윈도우 | 5개 영역, ①~④는 매번 반복 전송 |
| `/context` | 토큰 분포 진단, 수시로 점검 |
| `/compact` | 전체 또는 주제 집중 압축 선택 |
| `/clear` | 새 작업·주제 전환 시 초기화 |
| 긴 세션 | 자동 압축 임계값, 서브에이전트 격리 활용 |

컨텍스트 관리는 Claude Code 숙련도의 핵심입니다. 도구를 잘 쓰는 것도 중요하지만, 한정된 창 안에서 맥락을 유지하는 기술이 생산성을 결정합니다. `/context`로 진단하고, `/compact`로 압축하고, `/clear`로 전환하세요. 이 세 가지 슬래시 커맨드가 여러분의 세션을 안정적으로 지켜줄 것입니다.
