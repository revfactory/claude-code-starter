# 챕터 3: 첫 번째 대화 — Hello, Claude Code!

> **학습 목표**
> - 프로젝트 디렉토리에서 Claude Code를 시작할 수 있다
> - 자연어로 코드에 대해 질문할 수 있다
> - Claude Code가 파일을 읽고, 수정하고, 명령을 실행하는 흐름을 체험한다
> - 권한 승인 프로세스를 이해한다

---

설치를 마쳤으니 이제 직접 써볼 차례입니다. 이번 챕터에서는 샘플 프로젝트를 열고, Claude Code와 첫 대화를 나눠봅니다. 코드를 질문하고, 수정하고, 되돌리는 과정까지 한 번에 체험합니다.

---

## 3.1 프로젝트에서 시작하기

Claude Code는 **워크스페이스** 기반으로 동작합니다. 실행한 디렉토리의 파일을 컨텍스트로 읽어들이죠. 따라서 항상 프로젝트 루트에서 시작하는 것이 중요합니다.

먼저 실습용 프로젝트를 준비합니다.

### 코드 예제 1: 샘플 프로젝트 시작

```bash
$ mkdir hello-claude && cd hello-claude
$ git init
$ cat > app.js << 'EOF'
function greet(name) {
  return `Hello, ${name}!`;
}

function add(a, b) {
  return a + b;
}

module.exports = { greet, add };
EOF
$ git add -A && git commit -m "초기 커밋"
```

프로젝트가 준비되었으면 Claude Code를 시작합니다.

```bash
$ claude
```

터미널에 대화형 프롬프트가 나타납니다.

```
╭──────────────────────────────────╮
│ ✻ Welcome to Claude Code!        │
│                                  │
│   /hello-claude                  │
╰──────────────────────────────────╯

>
```

> **팁**: `claude "이 프로젝트가 뭐야?"` 처럼 초기 프롬프트를 함께 넘길 수도 있습니다. 빠르게 질문 하나만 던지고 싶을 때 유용합니다.

Claude Code는 실행 시점의 디렉토리를 워크스페이스로 인식합니다. `app.js`와 `.git` 이력 모두 컨텍스트로 활용할 수 있게 되죠.

> **주의**: 홈 디렉토리(`~`)에서 실행하면 수많은 파일이 컨텍스트 후보가 됩니다. 반드시 프로젝트 디렉토리로 이동한 뒤 시작하세요.

## 3.2 코드베이스에 질문하기

대화형 프롬프트에 자연어를 입력하면 됩니다. Claude Code가 알아서 파일을 읽고 분석합니다.

### 코드 예제 2: 코드베이스 질문 대화 예시

```bash
> 이 프로젝트가 무엇을 하나요?
```

```
[응답]
이 프로젝트는 두 가지 유틸리티 함수를 제공하는 Node.js 모듈입니다.

1. greet(name) — 이름을 받아 인사 문자열을 반환합니다.
2. add(a, b) — 두 숫자를 더해 결과를 반환합니다.

app.js 하나의 파일로 구성되어 있으며,
module.exports로 두 함수를 외부에 공개합니다.
```

이어서 더 구체적인 질문도 해봅니다.

```bash
> greet 함수에 기본값이 있나요?
```

```
[응답]
아니요, greet 함수에는 기본 매개변수가 설정되어 있지 않습니다.
name을 전달하지 않으면 "Hello, undefined!"가 반환됩니다.
기본값을 추가하고 싶으시면 말씀해주세요.
```

이것이 에이전틱 루프의 첫 단계입니다. Claude Code가 파일을 읽고(Context 수집), 분석 결과를 보여줍니다. 아직 파일을 수정하지 않았으므로 권한 승인이 필요하지 않았습니다.

> **팁**: 질문은 한국어든 영어든 상관없습니다. 다만 코드 관련 용어를 섞어 쓰면 더 정확한 답을 얻을 수 있습니다. 예를 들어 "greet 함수의 파라미터를 설명해줘"처럼요.

## 3.3 첫 코드 수정

이제 코드를 직접 수정해 봅시다. 자연어로 원하는 변경을 설명하면 됩니다.

### 코드 예제 3: 파일 수정 요청 및 승인 과정

```bash
> greet 함수에 기본값 "World"를 추가해줘
```

Claude Code가 수정 계획을 세우고, 파일 편집을 시도합니다. 이때 화면에 다음과 같은 승인 요청이 나타납니다.

```
Claude wants to edit app.js

  function greet(name) {          function greet(name = "World") {
    return `Hello, ${name}!`;       return `Hello, ${name}!`;
  }                               }

  Allow edit to app.js?
  [Y]es  [N]o  [A]lways allow edits
```

여기서 **Y**를 누르면 수정이 적용됩니다.

```
[응답]
app.js의 greet 함수에 기본 매개변수 "World"를 추가했습니다.
이제 greet()을 인자 없이 호출하면 "Hello, World!"를 반환합니다.
```

수정된 파일을 확인해 봅시다.

```bash
> app.js 내용을 보여줘
```

```
[응답]
function greet(name = "World") {
  return `Hello, ${name}!`;
}

function add(a, b) {
  return a + b;
}

module.exports = { greet, add };
```

깔끔하게 반영되었습니다. 이것이 에이전틱 루프의 두 번째 단계, Action 수행입니다.

## 3.4 권한 승인 이해하기

방금 경험한 승인 팝업을 좀 더 자세히 살펴봅시다.

Claude Code는 **안전 제일** 원칙을 따릅니다. 파일을 읽는 것은 자유롭지만, **수정**하거나 **명령을 실행**할 때는 반드시 여러분의 허락을 구합니다.

### 승인이 필요한 동작

| 동작 | 예시 | 승인 필요 |
|------|------|----------|
| 파일 읽기 | 코드 분석, 검색 | 아니오 |
| 파일 수정 | 함수 추가, 버그 수정 | **예** |
| 파일 생성 | 새 파일 작성 | **예** |
| 명령 실행 | `npm test`, `git commit` | **예** |

### 승인 선택지

승인 요청이 나타나면 다음 중 하나를 선택합니다.

- **Y (Yes)**: 이번 한 번만 허용합니다.
- **N (No)**: 거부합니다. Claude Code가 대안을 제안합니다.
- **A (Always)**: 이 세션에서 같은 유형의 동작을 항상 허용합니다.

예를 들어, 명령 실행 승인도 살펴봅시다.

```bash
> npm test를 실행해줘
```

```
Claude wants to run:
  npm test

  Allow?
  [Y]es  [N]o  [A]lways allow Bash commands
```

> **팁**: 반복적으로 승인하기 번거롭다면 **Shift+Tab**으로 권한 모드를 `acceptEdits`로 전환하세요. 파일 편집은 자동 승인되고, 명령 실행만 확인합니다. 권한 모드에 대해서는 챕터 10에서 자세히 다룹니다.

> **주의**: `bypassPermissions` 모드는 모든 승인을 건너뜁니다. 격리된 컨테이너 환경이 아니라면 절대 사용하지 마세요.

### 왜 이런 구조일까요?

AI가 자율적으로 코드를 수정하고 명령을 실행하는 것은 강력하지만, 동시에 위험할 수 있습니다. 잘못된 `rm -rf`나 의도치 않은 파일 덮어쓰기를 방지하려면, 사람이 최종 결정권을 가져야 합니다. 이것이 Claude Code의 **Human-in-the-loop** 설계 철학입니다.

## 3.5 되돌리기

실수는 언제든 일어납니다. Claude Code는 모든 파일 변경 전에 **체크포인트**를 자동으로 저장합니다. 되돌리기가 아주 쉽습니다.

### 코드 예제 4: 체크포인트 복원 실습

일부러 큰 변경을 요청해 봅시다.

```bash
> app.js의 모든 함수를 화살표 함수로 바꿔줘
```

```
Claude wants to edit app.js

  function greet(name = "World") {    const greet = (name = "World") =>
    return `Hello, ${name}!`;           `Hello, ${name}!`;
  }
                                      const add = (a, b) => a + b;
  function add(a, b) {
    return a + b;
  }

  Allow edit to app.js?
  [Y]es  [N]o  [A]lways allow edits
```

**Y**를 눌러 수정을 적용합니다.

```
[응답]
app.js의 모든 함수를 화살표 함수로 변환했습니다.
```

그런데 마음이 바뀌었습니다. 원래대로 돌리고 싶으면 어떻게 할까요?

**Esc 키를 2번** 누르세요.

```
Undo last change?
  app.js — Revert to previous checkpoint

  [Y]es  [N]o
```

**Y**를 누르면 수정 전 상태로 복원됩니다.

```
[응답]
app.js를 이전 체크포인트로 복원했습니다.
```

파일을 확인하면 화살표 함수 변환 전으로 돌아가 있습니다.

```bash
> app.js 내용 확인
```

```
[응답]
function greet(name = "World") {
  return `Hello, ${name}!`;
}

function add(a, b) {
  return a + b;
}

module.exports = { greet, add };
```

완벽히 복원되었습니다.

### 되돌리기 정리

| 방법 | 동작 | 사용 시점 |
|------|------|----------|
| **Esc 2번** | 마지막 변경을 체크포인트로 복원 | 방금 한 수정이 마음에 안 들 때 |
| **Ctrl+C** | 현재 진행 중인 작업 취소 | Claude Code가 작업 중일 때 멈추고 싶을 때 |

> **팁**: 체크포인트는 Git과 별개로 동작합니다. Git 커밋을 하지 않았어도 되돌릴 수 있습니다. 다만 세션이 종료되면 체크포인트는 사라지므로, 중요한 변경은 Git 커밋으로 저장하세요.

## 정리

이번 챕터에서 여러분은 Claude Code의 핵심 흐름을 체험했습니다.

1. **프로젝트 디렉토리에서 시작** — `cd project && claude`
2. **자연어로 질문** — Claude Code가 파일을 읽고 분석
3. **코드 수정 요청** — 승인 후 파일에 반영
4. **권한 승인** — Allow/Deny로 안전하게 제어
5. **되돌리기** — Esc 2번으로 체크포인트 복원

이 다섯 가지가 Claude Code 사용의 기본 골격입니다. 앞으로 배울 모든 기능은 이 흐름 위에 쌓여갑니다.
