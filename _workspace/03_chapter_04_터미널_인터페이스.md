# 챕터 4: 터미널 인터페이스 마스터하기

> Claude Code의 터미널을 자유자재로 다루는 법을 배웁니다.
> 단축키, 입력 기법, 세션 관리까지 실전 노하우를 익혀봅시다.

## 학습 목표

이 챕터를 마치면 여러분은 다음을 할 수 있습니다.

- 주요 단축키를 활용할 수 있다
- 멀티라인 입력과 이미지 붙여넣기를 사용할 수 있다
- 세션을 관리(재개, 이름 지정)할 수 있다
- 슬래시 커맨드의 기본 사용법을 안다

---

## 4.1 단축키 총정리

터미널 작업의 속도는 단축키가 결정합니다. Claude Code에서 제공하는 단축키를 표로 정리했습니다.

### 전체 단축키 표

| 단축키 | 기능 | 설명 |
|--------|------|------|
| **Ctrl+C** | 입력/생성 취소 | 현재 작업을 즉시 중단합니다 |
| **Shift+Tab** | 권한 모드 전환 | default → acceptEdits → plan 순환 |
| **Ctrl+L** | 화면 지우기 | 터미널 출력을 깨끗이 비웁니다 |
| **Ctrl+V / Cmd+V** | 이미지 붙여넣기 | 클립보드의 이미지를 입력합니다 |
| **!** (접두사) | bash 명령 실행 | Claude를 거치지 않고 직접 실행 |
| Ctrl+D | 세션 종료 | Claude Code를 종료합니다 |
| Ctrl+O | 상세 출력 전환 | 도구 실행 결과의 표시 수준 조절 |
| Ctrl+J | 라인 피드 | 입력 중 줄바꿈을 삽입합니다 |
| @ | 파일 경로 자동완성 | 파일 참조를 빠르게 입력합니다 |

### 가장 많이 쓰는 5가지

여러분이 매일 사용할 단축키 다섯 가지를 소개합니다.

**1. Ctrl+C — 즉시 멈춤**

Claude가 너무 긴 응답을 생성할 때, 또는 잘못된 방향으로 작업할 때 사용합니다. 입력 중에 누르면 현재 입력도 취소됩니다.

**2. Shift+Tab — 권한 모드 순환**

작업 성격에 따라 모드를 즉시 전환합니다. 코드를 살펴볼 때는 `plan`, 빠르게 수정할 때는 `acceptEdits`로 바꿔보세요.

**3. Ctrl+L — 화면 정리**

출력이 쌓여서 복잡해지면 화면을 깨끗이 비웁니다. 세션 데이터는 유지됩니다.

**4. Ctrl+V — 이미지 입력**

스크린샷이나 디자인 이미지를 붙여넣으면 Claude가 분석합니다. 에러 화면 캡처를 바로 전달할 때 유용합니다.

**5. ! — bash 바로 실행**

```bash
!git status
!ls -la src/
!npm test
```

프롬프트 앞에 `!`를 붙이면 Claude를 거치지 않고 터미널 명령을 바로 실행합니다. 빠른 확인 작업에 딱 맞습니다.

> **팁**: `Ctrl+C`는 두 가지 상황에서 동작합니다. 입력 중이면 입력을 취소하고, Claude가 응답 생성 중이면 생성을 중단합니다.

## 4.2 입력의 기술

### 멀티라인 입력

긴 프롬프트를 작성할 때 한 줄로는 부족합니다. Claude Code는 여러 가지 줄바꿈 방법을 지원합니다.

| 방법 | 지원 환경 | 비고 |
|------|-----------|------|
| `\` + Enter | 모든 터미널 | 가장 범용적 |
| Option+Enter | macOS 기본 터미널 | Mac 사용자 추천 |
| Shift+Enter | iTerm2, WezTerm, Ghostty, Kitty | 고급 터미널 |
| Ctrl+J | 모든 터미널 | 라인 피드 삽입 |

실제 사용 예시를 보겠습니다.

```bash
# \ + Enter로 멀티라인 입력
src/auth.ts 파일에서 \
로그인 함수의 입력값 검증을 추가하고 \
실패 시 에러 메시지를 한국어로 반환해줘
```

> **팁**: iTerm2나 Ghostty를 사용하신다면 `Shift+Enter`가 가장 자연스럽습니다. 일반 터미널에서는 `\` + Enter를 사용하세요.

### 이미지 붙여넣기

Claude Code는 이미지를 직접 이해합니다. 방법은 간단합니다.

1. 스크린샷을 찍거나 이미지를 복사합니다
2. Claude Code 프롬프트에서 `Ctrl+V`(또는 `Cmd+V`)를 누릅니다
3. 이미지가 첨부되었다는 표시가 나타납니다
4. 텍스트 프롬프트와 함께 전송합니다

```bash
# 이미지를 붙여넣은 후 입력
[이미지 첨부됨]
이 에러 화면을 분석해서 원인과 해결책을 알려줘
```

UI 디자인 시안을 붙여넣고 구현을 요청하거나, 에러 스크린샷으로 디버깅을 요청할 때 매우 편리합니다.

### @파일 참조

`@`를 입력하면 파일 경로 자동완성이 활성화됩니다. 특정 파일을 컨텍스트에 포함시킬 때 사용합니다.

```bash
@src/components/Button.tsx 이 컴포넌트에 loading 상태를 추가해줘
```

> **주의**: 이미지 붙여넣기는 터미널 에뮬레이터에 따라 지원 여부가 다릅니다. 동작하지 않으면 파일 경로를 직접 전달하세요.

## 4.3 슬래시 커맨드

슬래시 커맨드는 Claude Code의 내장 기능을 호출합니다. `/`로 시작하며, 대화 흐름을 관리하는 핵심 수단입니다.

### 필수 슬래시 커맨드 8가지

| 커맨드 | 기능 | 사용 시점 |
|--------|------|-----------|
| `/help` | 도움말 표시 | 커맨드가 기억나지 않을 때 |
| `/clear` | 세션 초기화 | 새로운 작업을 시작할 때 |
| `/cost` | 토큰 사용량 확인 | 비용을 점검할 때 |
| `/model` | 모델 변경 | 작업 난이도에 따라 전환 |
| `/compact` | 컨텍스트 압축 | 대화가 길어졌을 때 |
| `/context` | 컨텍스트 상태 확인 | 토큰 분포를 볼 때 |
| `/memory` | CLAUDE.md 보기/편집 | 프로젝트 규칙을 수정할 때 |
| `/resume` | 이전 세션 재개 | 중단된 작업을 이어갈 때 |

### 실전 활용 예제

**비용 확인과 모델 전환**

```bash
# 현재 비용 확인
/cost

# 결과 예시:
# Session tokens: 45,230 (input) / 12,100 (output)
# Estimated cost: $0.42

# 비용이 높으면 가벼운 모델로 전환
/model sonnet
```

**컨텍스트 관리**

```bash
# 컨텍스트 사용량 확인
/context

# 대화가 길어지면 압축
/compact

# 특정 주제에 집중하여 압축
/compact API 엔드포인트 변경사항만 유지해줘
```

> **팁**: `/compact`에 집중할 주제를 인자로 전달하면 해당 내용을 우선 보존합니다. 맥락이 흐려지는 것을 방지할 수 있습니다.

> **주의**: `/clear`는 세션 기록을 완전히 초기화합니다. 이전 대화를 다시 참조해야 한다면 `/compact`를 먼저 고려하세요.

## 4.4 세션 관리

Claude Code는 대화를 세션 단위로 관리합니다. 작업을 중단했다가 이어가는 것이 자유롭습니다.

### 세션 재개

```bash
# 가장 최근 세션 재개
claude -c

# 세션 목록에서 선택하여 재개
claude -r

# 특정 이름의 세션 재개
claude -r "auth-refactor"
```

### 세션 이름 지정

작업 중인 세션에 이름을 붙이면 나중에 쉽게 찾을 수 있습니다.

```bash
# 세션 중 이름 변경
/rename auth-refactor

# 이름으로 바로 재개
claude -r "auth-refactor"
```

### 세션 관리 워크플로우

실제 개발에서 세션을 활용하는 흐름을 보겠습니다.

```bash
# 1. 새 작업 시작
claude
> 로그인 기능을 리팩토링해줘

# 2. 작업 중 이름 지정
/rename login-refactor

# 3. 퇴근 — Ctrl+D로 종료

# 4. 다음 날 이어서 작업
claude -r "login-refactor"
> 어제 작업 이어서 테스트 코드도 추가해줘
```

> **팁**: 여러 기능을 병렬로 개발할 때 세션마다 이름을 붙여두면 전환이 수월합니다. `claude -c`는 가장 최근 세션만 재개하므로, 여러 세션을 오갈 때는 `claude -r`을 사용하세요.

## 4.5 Vim 모드와 테마

### Vim 모드

Vim 키바인딩에 익숙한 여러분을 위한 기능입니다.

```bash
# Vim 모드 활성화/비활성화 토글
/vim
```

활성화하면 프롬프트 입력창에서 Vim의 일반 모드, 입력 모드를 사용할 수 있습니다. `Esc`로 일반 모드, `i`로 입력 모드에 진입합니다.

### 테마 변경

터미널의 시각적 스타일을 바꿀 수 있습니다.

```bash
# 테마 선택 화면 열기
/theme
```

화살표 키로 원하는 테마를 선택하면 즉시 적용됩니다. 밝은 배경 터미널에서는 라이트 테마를, 어두운 배경에서는 다크 테마를 추천합니다.

> **팁**: Vim 모드와 테마 설정은 세션이 바뀌어도 유지됩니다. 한 번만 설정하면 됩니다.
