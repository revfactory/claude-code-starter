# 챕터 1: Claude Code란 무엇인가

---

## 이 챕터에서 배울 것

- Claude Code가 무엇인지 한 문장으로 설명할 수 있다
- 기존 AI 코딩 도구(Copilot 등)와 차이를 이해한다
- 에이전틱(Agentic) 코딩의 개념을 파악한다
- Claude Code로 할 수 있는 일의 범위를 안다

---

## 1.1 AI 코딩 도구의 진화

AI 코딩 도구는 세 단계를 거쳐 발전했습니다.

### 1세대: 자동완성

여러분이 코드를 타이핑하면, AI가 다음 줄을 제안합니다. 한 줄, 또는 한 블록 단위입니다. 현재 파일만 보고, 제안만 합니다. 여러분이 직접 수락하거나 무시해야 합니다.

### 2세대: 채팅

IDE 안에 채팅 창이 생겼습니다. 코드에 대해 질문하고, 답변을 받습니다. "이 함수를 리팩토링해줘"라고 요청하면 수정된 코드를 보여줍니다. 하지만 여전히 여러분이 직접 복사해서 붙여넣어야 합니다.

### 3세대: 에이전트

Claude Code가 바로 이 단계입니다. 질문에 답하는 것을 넘어, 직접 행동합니다. 파일을 읽고, 수정하고, 명령어를 실행합니다. 테스트를 돌리고, 결과를 확인합니다. 실패하면 스스로 수정하고 다시 시도합니다.

```
┌─────────────────────────────────────────────────────┐
│          AI 코딩 도구의 진화                          │
│                                                     │
│  1세대          2세대           3세대                 │
│  자동완성   →   채팅 보조   →   에이전트              │
│                                                     │
│  ∙ 한 줄 제안    ∙ 대화형 Q&A    ∙ 자율적 행동        │
│  ∙ 현재 파일만   ∙ 코드 생성     ∙ 전체 프로젝트      │
│  ∙ 수동 적용     ∙ 수동 적용     ∙ 자동 적용+검증     │
└─────────────────────────────────────────────────────┘
```

핵심은 **자율성**입니다. 1세대와 2세대는 여러분의 손이 필요합니다. 3세대 에이전트는 여러분의 지시만 있으면 스스로 움직입니다. 이것이 에이전틱 코딩의 본질입니다.

---

## 1.2 Claude Code의 핵심 특징

Claude Code를 한 문장으로 정의하면 이렇습니다.

> **Claude Code는 터미널에서 실행되는 에이전틱 AI 코딩 도구로, 전체 프로젝트를 이해하고 자율적으로 코드를 읽고, 수정하고, 검증합니다.**

이제 핵심 특징 네 가지를 살펴보겠습니다.

### 전체 코드베이스 이해

기존 도구는 열려 있는 파일 하나만 봅니다. Claude Code는 다릅니다. 프로젝트의 모든 파일을 읽고 분석합니다. 폴더 구조, 의존성, 설정 파일까지 파악합니다. "이 프로젝트가 무엇을 하나요?"라고 물으면, 전체 아키텍처를 설명해줍니다.

### 자율적 작업 수행

"로그인 함수에 입력 검증을 추가해줘." 이 한마디면 Claude Code가 알아서 움직입니다. 관련 파일을 찾고, 코드를 수정하고, 필요하면 새 파일도 만듭니다. 여러 파일에 걸친 변경도 한 번에 처리합니다.

### 다중 환경 지원

터미널이 기본이지만, 그것만은 아닙니다. VS Code, JetBrains IDE에서도 사용할 수 있습니다. 데스크톱 앱, 웹, 심지어 Slack에서도 동작합니다. 여러분이 편한 환경을 선택하면 됩니다.

### 대화형 상호작용

Claude Code는 독주하지 않습니다. 작업 중간에 언제든 멈출 수 있습니다. 방향을 바꿀 수도 있습니다. "잠깐, 그 방식 말고 다른 방법으로 해줘." 이런 요청도 자연스럽게 처리합니다.

> **팁**: Claude Code의 가장 큰 장점은 "맥락"입니다. 전체 프로젝트를 이해하기 때문에, 여러분이 일일이 설명하지 않아도 됩니다.

---

## 1.3 에이전틱 루프

Claude Code가 작업하는 방식을 들여다보겠습니다. 세 단계가 반복되는 루프 구조입니다.

```
        ┌──────────────────┐
        │  1. Context 수집  │
        │  파일 읽기, 검색   │
        │  상태 파악         │
        └────────┬─────────┘
                 │
                 ▼
        ┌──────────────────┐
        │  2. Action 수행   │
        │  파일 편집         │
        │  명령 실행         │
        └────────┬─────────┘
                 │
                 ▼
        ┌──────────────────┐
        │  3. 검증          │◄───┐
        │  테스트 실행       │    │
        │  결과 확인         │    │
        └────────┬─────────┘    │
                 │   실패 시     │
                 └──────────────┘
                    성공 시 ▼
                 ┌──────────┐
                 │   완료    │
                 └──────────┘
```

### 1단계: Context 수집

Claude Code는 먼저 상황을 파악합니다. 관련 파일을 읽습니다. 프로젝트 구조를 탐색합니다. 코드 패턴을 검색합니다. 충분한 컨텍스트를 모은 뒤에야 행동합니다.

### 2단계: Action 수행

컨텍스트를 바탕으로 행동합니다. 파일을 편집하거나, 새로 생성합니다. 필요하면 셸 명령도 실행합니다. Git 작업도 수행합니다.

### 3단계: 검증

행동한 뒤에는 반드시 확인합니다. 테스트를 실행합니다. 결과를 분석합니다. 실패하면 원인을 파악하고, 다시 2단계로 돌아갑니다. 성공할 때까지 이 루프를 반복합니다.

> **팁**: 이 루프가 Claude Code를 단순한 코드 생성기와 구분짓는 핵심입니다. "만들고 끝"이 아니라 "만들고, 확인하고, 고치는" 과정을 자동으로 수행합니다.

이것이 에이전틱 코딩입니다. 여러분은 목표를 제시하고, Claude Code는 스스로 루프를 돌며 목표를 달성합니다.

## 1.4 할 수 있는 일들

Claude Code의 활용 범위는 넓습니다. 대표적인 작업들을 살펴보겠습니다.

### 버그 수정

에러 메시지를 붙여넣으면 됩니다. Claude Code가 원인을 추적합니다. 관련 코드를 찾아 수정합니다. 수정 후 테스트까지 돌려줍니다.

### 기능 구현

"사용자 프로필 페이지를 만들어줘." 자연어 한 문장이면 충분합니다. 여러 파일에 걸쳐 코드를 생성합니다. 라우팅, 컴포넌트, 스타일까지 한 번에 만듭니다.

### 코드 리팩토링

전체 프로젝트의 아키텍처를 개선합니다. 중복 코드를 통합합니다. 네이밍 컨벤션을 통일합니다. 여러 파일에 걸친 대규모 변경도 처리합니다.

### 테스트 작성과 실행

"이 함수의 테스트를 작성해줘." 테스트 코드를 생성하고, 직접 실행합니다. 실패하는 케이스가 있으면 코드를 수정합니다. 모든 테스트가 통과할 때까지 반복합니다.

### Git 작업

변경사항을 분석해 커밋 메시지를 자동 생성합니다. PR(Pull Request)도 만들어줍니다. 브랜치 관리, 코드 리뷰 보조까지 가능합니다.

### 문서화

README 작성, API 문서 생성을 도와줍니다. 코드 주석도 추가합니다. 프로젝트의 실제 코드를 기반으로 정확한 문서를 만듭니다.

> **주의**: Claude Code는 강력하지만, 만능은 아닙니다. 최종 결과물은 반드시 여러분이 검토해야 합니다. AI는 도구이고, 의사결정은 개발자의 몫입니다.

다음 표는 Claude Code와 기존 자동완성형 AI 도구의 차이를 정리한 것입니다.

| 비교 항목 | 자동완성형 AI (Copilot 등) | Claude Code |
|----------|--------------------------|-------------|
| **작업 범위** | 현재 파일의 다음 줄 제안 | 전체 프로젝트 이해 및 다중 파일 수정 |
| **자율성** | 각 제안마다 개별 승인 필요 | 지시에 따라 여러 단계 자동 실행 |
| **도구 접근** | 코드 제안만 가능 | 파일 편집, 명령 실행, Git 작업, 검색 |
| **검증 능력** | 검증 불가 | 테스트 실행, 결과 확인, 자동 오류 수정 |
| **확장성** | 제한적 | MCP 서버, 스킬, 훅, 커스텀 에이전트 |
| **상호작용** | 자동완성 수락/거절 | 자연어 대화, 중간 방향 전환 가능 |

## 정리

이 챕터의 핵심을 요약합니다.

1. **Claude Code의 정의**: 터미널 기반 에이전틱 AI 코딩 도구입니다. 전체 프로젝트를 이해하고, 자율적으로 코드를 읽고, 수정하고, 검증합니다.

2. **AI 코딩 도구의 진화**: 자동완성(1세대) → 채팅(2세대) → 에이전트(3세대)로 발전했습니다. Claude Code는 3세대 에이전트입니다.

3. **에이전틱 루프**: Context 수집 → Action 수행 → 검증의 3단계 루프를 반복합니다. 실패하면 스스로 수정하고 다시 시도합니다.

4. **핵심 특징 네 가지**: 전체 코드베이스 이해, 자율적 작업 수행, 다중 환경 지원, 대화형 상호작용입니다.

5. **활용 범위**: 버그 수정, 기능 구현, 리팩토링, 테스트, Git 작업, 문서화 등 개발의 거의 모든 영역을 지원합니다.
